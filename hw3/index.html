Overview

In this homework, we implemented a path tracer that makes it possible for our rendering to simulate the behavior of light in real life. We first begin by generating rays from a virtual camera and detecting intersections with primitives. Then to handle more complex meshes, I implemented the bounding volume hierarchy by recursively partitioning a scene into axis aligned bounding boxes that can significantly reduce the amount of time it takes to render a scene. After that, we implemented the hemisphere and importance direct lighting methods and noted the differences between those renderings. Then we implemented the path tracing that would allow us to simulate indirect lighting, or light that bounces off multiple surfaces before reaching the camera. Then we finished off by looking to see if rays would converge and, if so, would stop rendering those pixels. Through this project I was able to connect the knowledge I learned in lecture to the actual code required to render these images. It was also fun to see the difference between renderings through each stage and also when the parameters of number of light rays or number of samples would adjust.
